# ADR 0005: Platform data and event services

- Status: Accepted
- Date: 2026-07-22

## Context

UWO products need shared object metadata, reliable notification delivery, privacy-safe operational analytics, and durable audit evidence. These capabilities must preserve the Phase 3A identity boundary and Phase 3B transactional/idempotency standards without selecting cloud storage, messaging vendors, brokers, or production databases. Prompts, outputs, file bytes, message bodies, credentials, and payment data must not leak into cross-service events.

## Decision

Create separate `platform_storage`, `platform_notifications`, `platform_analytics`, and `platform_audit` services with schema-versioned canonical contracts in `packages/contracts`. Each service owns repository protocols and a UnitOfWork. Committed thread-safe in-memory repositories implement rollback for tests only; production startup refuses to instantiate them.

Storage is metadata-only. Raw bytes remain behind an injected `BlobStore`; callers never choose storage keys. Object versions append immutably and bind content length, SHA-2 digest, tenant, product, region, and creator. Malware results form a separate append-only history; deterministic current scan status is derived without rewriting an object version. Expiry and retention are compared as parsed UTC instants. Restricted/regulated retention must be future-dated, and shortening requires a reasoned, audited platform-admin override. Download authorization contains an opaque token with bounded positive TTL, not a provider URL or credential, and fails closed when region, retention, legal-hold, scan, or integrity policy is unavailable.

Notification templates are immutable versions, numbered from the maximum existing version. Creation of an eligible notification and its delivery outbox record commits atomically after template product, region, and channel checks. Delivery follows claim-deliver-finalize: claim and lease transactionally, commit, call the provider outside the UnitOfWork with a stable idempotency key, then transactionally record attempt and terminal/retry state. Expired leases recover; retry due time is enforced; exact accepted replay does not create another external delivery; cancellation and every terminal notification resolve the outbox. Conflicting deduplication reuse fails closed, bounded exponential retry has no sleep dependency, and permanent/exhausted failures create a dead letter.

Analytics events are append-only and globally unique. Their contract contains only tenant, product, region, an allowlisted operational event type, fixed outcome/bucket/error dimensions, UTC time, and optionally an approved pseudonymous subject. Arbitrary metadata and sensitive content have no field. The service replaces caller `recorded_at`, enforces lateness and future-skew limits, and uses parsed half-open `[start, end)` UTC windows. Metric identities bind all dimensions and boundaries. Immutable snapshot hashes preserve reproducibility; minimum-count suppression applies to every export, and platform-admin cross-tenant output contains aggregate points only.

Audit uses append-only tenant streams with transactionally allocated monotonic sequence numbers. Every current SHA-256 hash binds the prior hash and canonical allowlisted scalar event content. Checkpoints verify the complete contiguous chain from genesis and reject missing/duplicate sequence positions. Range export verifies the full chain, exact requested boundaries, and range continuity; manifests hash canonical exported event content. Source-event IDs and fingerprints make audit publishing idempotent and conflicting reuse fail closed. Pseudonymous source provenance is preserved separately from authenticated direct-append identity. Audit events never retain request bodies, content, credentials, stack traces, or arbitrary exceptions.

Use provider-neutral deterministic `PlatformEvent` records to integrate the gateway, control plane, billing, and Phase 3C services. Event validation requires stable identifiers, UTC, a supported schema, deeply immutable allowlisted scalar attributes, and an ID fingerprint covering type, tenant, request, and safe attributes. Where a business mutation creates downstream work, including a tenant-status change, the immutable outbox record shares its transaction. Dispatchers take exclusive expiring leases, enforce due times and maximum attempts, recover abandoned work, preserve event identity, and acknowledge only after idempotent downstream acceptance. Duplicate acknowledgement is idempotent; cancellation and terminal business state terminally resolve work; poison events become visible dead letters.

Provider execution keeps an injected durable-outcome contract separate from event payloads. It records the redacted provider result required to resume billing capture, marks capture completion, and then records a safe success event. An event delivery failure cannot convert a successful capture into a request failure, and exact replay after process recreation cannot call the provider or charge twice. The committed implementation is in-memory for tests; production must supply durable workflow/outbox storage. Services without event integration use a null recorder.

Application services own successful mutation and rollback audit events; rollback is emitted only after mutation begins. HTTP boundaries own exactly one rejected-request event and one unexpected-internal-error event, map provider/infrastructure failures separately from policy denials, and return only stable redacted errors.

## Consequences

Tests can prove storage/scan immutability, retention governance, notification acceptance recovery without duplicate delivery, aggregation privacy, full-chain audit tamper detection, transactional rollback, lease exclusion, restart dispatch, and provider/capture deduplication without credentials or live providers. Production enablement still requires regional durable repositories, workflow outcomes, outboxes and blob storage, encryption and workload identity, approved malware/retention/legal-hold systems, real notification adapters, dispatch/reconciliation workers, integrity monitoring, backup/recovery, and Phase 3D telemetry/SLO/runbooks. This decision does not deploy infrastructure or store sensitive content in platform events, analytics, or audit.
